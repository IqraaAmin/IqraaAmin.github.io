Studying Object Oriented Programming (OOP) and Database Management Systems (DBMS) in the same semester was both challenging and rewarding. At first, they seemed like two completely different subjects. OOP focused on writing structured and reusable code, while DBMS focused on storing, organizing, and retrieving data efficiently. However, as the semester progressed, I began to realize that these subjects are closely connected and together form the foundation of modern software development.

Before this semester, I mostly viewed programming as writing code that performs specific tasks. I did not think much about how applications manage large amounts of information behind the scenes. Learning OOP and DBMS together helped me see the bigger picture. Software is not just about code or data alone—it is about how both work together to create useful and efficient systems.

## Understanding the Connection

One of the most interesting realizations during the semester was discovering how naturally OOP and databases complement each other. In OOP, we learn to represent real-world entities using classes and objects. In DBMS, we learn how information related to those entities can be stored and managed in a database.

For example, a student management system can be designed using OOP concepts such as classes for students, courses, and instructors. At the same time, the actual information about those students and courses is stored inside database tables. Understanding this relationship helped me appreciate how software applications function beyond the code that users see on the screen.

Initially, I struggled to connect these ideas. OOP introduced concepts such as encapsulation, inheritance, and polymorphism, while DBMS introduced normalization, relationships, and SQL queries. They felt like separate worlds with different rules and objectives. Over time, however, assignments and practical exercises showed me how these concepts work together to solve real-world problems.

The more examples I worked on, the clearer the connection became. OOP provided the structure and logic, while databases provided reliable data storage and retrieval. Together, they created complete solutions rather than isolated pieces of technology.

## Learning Through Practice

The practical sessions were where these concepts truly came to life. Writing SQL queries and designing database schemas required a different kind of thinking compared to writing code. Instead of focusing on program flow, I had to think about relationships between data, efficiency, and organization.

At the same time, OOP assignments challenged me to design programs that were clean, reusable, and easy to maintain. Sometimes a problem required careful planning before writing a single line of code. Other times, debugging became the biggest challenge.

There were moments when both subjects felt overwhelming. Database normalization rules seemed complicated at first, and OOP concepts such as inheritance required a significant shift in mindset. However, each challenge gradually improved my understanding and confidence.

One thing I particularly enjoyed was seeing how concepts from one subject supported learning in the other. The structured thinking developed through OOP helped me design better database-related projects, while understanding data relationships in DBMS improved my ability to model real-world systems in code.

As the semester continued, I became more comfortable working with both technologies. What initially seemed difficult eventually became enjoyable because I could see their practical value in software development.

## Why Both Subjects Matter

By the end of the semester, I understood why OOP and DBMS are considered fundamental subjects in Computer Engineering and Software Development. Nearly every modern application depends on both. Whether it is a social media platform, an online store, a banking system, or a university portal, software relies on well-structured code and efficient data management.

Learning these subjects together gave me a stronger foundation for future courses and projects. More importantly, they changed the way I think about building software. Instead of viewing applications as simple programs, I began seeing them as systems where code and data work together to solve real problems.

Looking back, studying OOP and DBMS in the same semester was demanding, but it was also one of the most valuable learning experiences of my academic journey. The knowledge gained from these subjects continues to influence how I approach programming and software design today.
